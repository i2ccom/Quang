- agent memories
- datasets

### Core idea

Large data should not be stored as flat bytes only.

It should be stored as:

```text
coarse structure → medium detail → fine detail → optional refinement
```

This enables:

- fast preview
- cheap compression
- progressive loading
- AI-friendly attention
- spatial locality
- multi-resolution reasoning
- efficient cache invalidation
- “zoomable” knowledge navigation

### Why it matters

AI does not always need full resolution.

An agent reading a codebase, image, video, or enterprise dataset often needs:

1. rough shape first,
2. important regions next,
3. exact bytes only when necessary.

Fractal makes data naturally compatible with how AI attention works.

### Pitch phrase

## Pillar 2: **Fluid**

### Tagline

**A next-generation Git-compatible data/file system for AI agents.**

### What it is

Fluid is the versioned storage format and data structure family.

It includes:

- **FluidArray**
- **FluidBlob**
- **FluidGraph**
- **FluidZip**
- **FluidFile**
- **FluidRepo**

Fluid is not just a ZIP of folders.

It is a smart, locality-aware, diffable container where data can be:

- compressed or uncompressed
- hot or cold
- cached or archived
- text or binary
- graph or array
- immutable or transactional
- AI-indexed
- partially loaded
- version-swapped
- partitioned
- streamed over disk or cloud

### Why Fluid beats classic Git

Git was revolutionary for source code, but it struggles with AI-era workloads:

| Git | Fluid |
|---|---|
| File/folder oriented | Graph/data/object oriented |
| Strong for text diffs | Strong for text, binary, graph, blob, array diffs |
| Poor large binary handling | Native binary delta and blob management |
| Snapshot mindset | Multiversion, partition, hot/cold lifecycle |
| Human developer first | Human + AI agent first |
| Folder checkout | JIT partial loading |
| Context externalized | Embedded indexes, graph, metadata, semantic maps |
| Code-centric | Code, data, UI, model, prompt, memory, media |

### Fluid as a file

A `.fluid` object can contain:

```text
code
assets
schemas
binary blobs
graph metadata
semantic indexes
vector hints
version history
transactions
AI summaries
dependency graph
hot/cold cache layout
```

So instead of:

```text
project.zip
folder/
  src/
  assets/
  docs/
  config/
```

You get:

```text
project.fluid
  FluidArray
  FluidBlob
  FluidGraph
  FluidIndex
  FluidDelta
  FluidManifest
  FluidMemory
```

### Incentive to migrate from Git

Fluid does not need to destroy Git.

It can start as:

- Git-compatible backend
- Git packfile accelerator
- binary delta layer
- AI context index
- local cache
- monorepo optimizer
- agent collaboration layer
- QuangHub repository format

Developers can still use Git commands, but Fluid provides a faster, smarter substrate.

### Pitch phrase

> **Fluid is Git reimagined for AI agents, binary worlds, massive repos, and living data.**
